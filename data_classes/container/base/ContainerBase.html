Parameters

  • dict_in ‚Äì the dictionary the container should wrap around. Default is None.

  • queues ‚Äì Sequence of multiprocessing queues, each of which returns containers. This enables the current container to be passed around asynchronously while waiting for data. Default is None.

  • queue_load_sizes ‚Äì Size of leading dimension of the containers returned by each queue. Default is None.

  • container_combine_method ‚Äì The method to use for combining containers arriving from different queues. Default is ivy.Container.list_join

  • queue_timeout ‚Äì The timeout when waiting for containers to arrive from the queues. Default is global.

  • print_limit ‚Äì The total array size limit when printing the container. Default is 10.

  • key_length_limit ‚Äì The maximum key length when printing the container. Default is None.

  • print_indent ‚Äì The number of whitespaces to use for indenting when printing the container. Default is 4.

  • print_line_spacing ‚Äì The number of extra newlines to use between keys when printing the container. Default is 0.

  • ivyh ‚Äì Handle to ivy module to use for the calculations. Default is None, which results in the global ivy.

  • default_key_color ‚Äì The default key color for printing the container to the terminal. Default is ‚Äògreen‚Äô.

  • keyword_color_dict ‚Äì A dict mapping keywords to their termcolor color codes for printing the container. (Default value = None)

  • rebuild_child_containers ‚Äì Whether to rebuild container found in dict_in with these constructor params. Default is False, in which case the original container are kept as are.

  • types_to_iteratively_nest ‚Äì The data types to nest iteratively in the dict structure, each type must be iterable. Default is None.

  • alphabetical_keys ‚Äì Whether to sort the container keys alphabetically, or preserve the dict order. Default is True.

  • kwargs ‚Äì keyword arguments for dict creation. Default is None.
